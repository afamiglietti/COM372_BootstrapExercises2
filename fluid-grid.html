The Fluid Grid

All grid elements must be nested inside a container. We usually only use one container on a page, and we never nest containers inside each other. This page uses the 'container-fluid' class, which creates a fluid design, taking up the full width of the page. Because the design is fluid (widths are defined in percent of screen, not pixels) it can fit the exact size of the browser. Fluid designs are powerful, but can also create challenges in maintaining alignment and sizing.
The "row" class contains the columns. We usually don't put content in a row without nesting it inside a column first, because the row has a negative margin.

Column Width 4

This is a column with a width of four. It has padding to keep it from crowding the next column. It has a width of 33.3%, which makes it take up a third of it's container. It doesn't change size in the fixed-width display (except at responsive breakpoints) because the container class has a fixed pixel width.

Fixed Height

I set a fixed height in pixels for this div, to demonstrate what can go wrong when fixed-heights and fluid design mix. As we resize the page the content "overflows" the fixed height container, with ugly results! (Note: You cannot set height in percent, only pixels.)
